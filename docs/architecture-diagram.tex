\caption{Architecture of the distributed traffic booking service.
  Three independent regional clusters (Laos, Cambodia, Andorra) each contain a route
  service, validation service (two instances), data service (two instances), authority
  service, Redis instance, and MongoDB instance.
  Global services --- booking service, journey management (with Redis leader election),
  and notification service --- run as two instances each behind NGINX and interact with
  all three clusters via Apache Kafka.}
